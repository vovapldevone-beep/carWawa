# Implementation Plan: Hover-підсвітка зон на мапі

## Контекст
- Проект на Laravel + Vue.
- Потрібно додати підсвітку існуючих зон (кіл) на Google Map при наведенні.
- Підсвітка: зелений колір.
- Підсвічується тільки коло, без змін для маркерів.
- Паралельно потрібно додати загальні правила для Laravel/Vue у `.cursor/rules`.
- Потрібно додати документацію у `README.md` з лінком на сторінку в `doc`.
- Вести короткі нотатки в `.cursor/rules/projects.mdc`.

## Цілі
1. Стандартизувати роботу з Laravel/Vue через локальні Cursor rules.
2. Реалізувати UX-покращення: hover-виділення зон на мапі.
3. Задокументувати фічу в `doc` і дати доступ до неї через `README.md`.

## Stages

### Stage 1: Додати загальні правила Laravel/Vue
**Файли:**
- `.cursor/rules/laravel-general-rules.mdc` (new)
- `.cursor/rules/vue-general-rules.mdc` (new)
- `.cursor/rules/projects.mdc` (update)

**Що робимо:**
- Додаємо загальні правила українською мовою:
  - структура коду, валідація, API-відповіді, безпека, тести (Laravel);
  - композиційний API, реактивність, стейт, продуктивність, робота з map-компонентами (Vue).
- Коротко фіксуємо виконання у `projects.mdc`.

### Stage 2: Реалізувати hover-виділення кіл у Vue-компоненті
**Файли:**
- `resources/js/components/ExampleComponent.vue` (update)
- `.cursor/rules/projects.mdc` (update)

**Що робимо:**
- Додаємо стан активної зони (ID зони в hover).
- Вмикаємо інтерактивність для кіл існуючих зон.
- Додаємо обробники hover-подій (наведення/вихід).
- Змінюємо `fillColor` та `strokeColor` на зелений для активної зони.
- Не змінюємо поведінку маркерів.
- Коротко фіксуємо виконання у `projects.mdc`.

### Stage 3: Додати документацію фічі і оновити README
**Файли:**
- `doc/map-zones-hover.md` (new)
- `README.md` (update)
- `.cursor/rules/projects.mdc` (update)

**Що робимо:**
- Створюємо документацію фічі українською:
  - опис,
  - поведінка,
  - короткі технічні деталі,
  - можливі майбутні розширення.
- Додаємо в `README.md` секцію `Features` з пунктом:
  - назва фічі;
  - лінк на `doc/map-zones-hover.md`.
- Коротко фіксуємо виконання у `projects.mdc`.

## Verification Plan
1. Перевірити, що нові rule-файли існують у `.cursor/rules`.
2. Запустити фронтенд і вручну перевірити:
   - при наведенні на будь-яке збережене коло воно стає зеленим;
   - при відведенні курсора колір повертається до базового;
   - маркери не змінюють стиль при hover.
3. Перевірити наявність:
   - `doc/map-zones-hover.md`,
   - секції `Features` у `README.md` з коректним лінком.
4. Перевірити, що в `projects.mdc` додані короткі записи по всіх виконаних кроках.

## Ризики та нотатки
- Для hover подій кола мають бути `clickable: true`, інакше події наведення можуть не спрацювати.
- Якщо мапа має багато зон, у майбутньому можна оптимізувати рендер (кластеризація/віртуалізація/спрощення стилів).
